# Contributing to Awesome Industrial Modbus

Thanks for considering a contribution!

## Quality bar

Before submitting, make sure your addition meets these criteria:

1. **Actively maintained** — commit within the last 12 months. Dead projects get removed during periodic audits.
2. **Clear license** — open-source preferred but well-known commercial tools OK if they are industry-standard (e.g., Modbus Poll, diagslave).
3. **Factual description** — one sentence describing what the tool does, no marketing language. Bad: "the ultimate Modbus solution". Good: "Modbus RTU/TCP slave simulator with TCP and Serial modes".
4. **Alphabetical order** within each section.
5. **One PR = one logical addition**. Dont bundle unrelated entries.

## How to PR

1. Fork this repo
2. Edit `README.md` in the appropriate section
3. Open a PR with title `Add: <ProjectName> to <Section>`
4. PR body: 1-2 sentences explaining why it fits the section

## Conflict of interest

If you maintain the project you are adding, please include a disclosure in the PR body. Disclosure does NOT prevent inclusion — projects are evaluated on technical merit. We just want transparency.

The maintainer of this list ([leaberg69](https://github.com/leaberg69)) discloses ownership of `aem-modbus-simulator`, `aem-modbus-cli`, and `aem-iec62443-checklist`. Competing alternatives are explicitly welcomed.

## Removing entries

If you find a dead link, broken project, or entry that no longer fits:
- Open a PR removing it with rationale
- Or open an issue and a maintainer will review

## License of contributions
